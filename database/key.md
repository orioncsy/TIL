# Key

## 개념

### key

- 검색, 정렬 시 레코드를 구분할 수 있는 속성

## 종류

### 후보키

- 레코드를 구분할 수 있는 속성들로 구성되어 있는 부분집합
- 기본키로 사용할 수 있는 속성들
- 유일성
    - key를 통해 하나의 레코드를 유일하게 식별 가능
- 최소성
    - 필요한 최소한의 속성들로 구성
- 유일성과 최소성을 모두 만족하는 키

### 기본키

- 레코드를 구분할 수 있는 속성들로 구성된 키
- null 값을 가질 수 없다.
- 동일한 값이 중복될 수 없다.

### 대체키

- 후보 키 중에서 기본키를 제외한 키

### 슈퍼키

- 유일성은 만족하지만 최소성을 만족하지 못하는 키

### 외래키

- 다른 테이블의 기본키를 참조하는 속성
